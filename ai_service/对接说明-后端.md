# 智能服务集成说明（给后端）

`ai_service` 是一个独立的 FastAPI 子模块（智能服务模块）。后端只需做**两件事**：
1. 把它的路由**挂到主后端**上；
2. 把图书数据的**取数逻辑换成查数据库**。

其它逻辑（调大模型、摘要/推荐/问答、失败兜底）都封装好了，不用动。

## 一、集成三步

### 1. 装依赖
`ai_service` 的依赖（fastapi/uvicorn 可能已有，openai、python-dotenv 需要补）：
```bash
pip install -r ai_service/requirements.txt
```

### 2. 配置 Key
```bash
cp ai_service/.env.example ai_service/.env
# 编辑 ai_service/.env 填 DASHSCOPE_API_KEY（阿里云百炼的 Key）
```
> `.env` 已在 `.gitignore` 里，**不会进 git**，各自本地配自己的 Key。

### 3. 挂载路由（核心，两行）
在后端主程序里：
```python
from fastapi import FastAPI
from ai_service.router import router as ai_router

app = FastAPI()
# ...自己的路由...
app.include_router(ai_router)      # ← 挂上后，就多了 /api/ai/* 三个接口
```

挂载后自动拥有：
| 接口 | 功能 |
|---|---|
| `POST /api/ai/summary` | 图书摘要 |
| `POST /api/ai/recommend` | 智能推荐 |
| `POST /api/ai/chat` | 多轮问答 |

> 前缀 `/api/ai` 已在 router 里定义好，不用再加。

## 二、接入真实图书数据（关键对接点）

**推荐功能**需要"从哪些书里挑"。现在读的是样例 JSON，只要改 **一个文件一个函数**，就换成查数据库——`ai_service/services/` 里的逻辑完全不用动。

改 `ai_service/book_repo.py` 的 `get_all_books()`：

```python
# ===== 现在（样例数据）=====
def get_all_books():
    with open(_DATA_PATH, encoding="utf-8") as f:
        return json.load(f)

# ===== 换成（查数据库）=====
def get_all_books():
    books = db.query(Book).all()          # 用实际的 ORM / 查询方式
    return [
        {
            "id": b.id,
            "title": b.title,
            "author": b.author,
            "category": b.category,        # 分类，如"科幻小说"
            "intro": b.description,        # 图书简介/描述
        }
        for b in books
    ]
```

**只要返回的每个 dict 是下面这个结构就行**，上层不用改。

## 三、图书数据结构契约

`get_all_books()` 返回 `list[dict]`，每个 dict：

| 字段 | 类型 | 说明 | 是否必需 |
|---|---|---|---|
| `id` | int | 图书 ID | 必需（推荐结果要回传 id） |
| `title` | str | 书名 | 必需 |
| `author` | str | 作者 | 建议有 |
| `category` | str | 分类 | 建议有（影响推荐质量） |
| `intro` | str | 简介/描述 | 建议有（推荐靠它理解内容） |

> 字段名对不上也没关系——在 `get_all_books()` 里把数据库字段映射成上面的名字即可（如上例把 `b.description` 映射成 `intro`）。

## 四、注意事项

- **只有推荐**用到图书数据；摘要、问答不碰数据库，不用管。
- `book_repo.py` 里有 `@lru_cache`（缓存），换成查库后如果图书会频繁更新，去掉这个缓存装饰器即可。
- CORS：`ai_service/main.py` 里的跨域是给**独立调试**用的；正式集成到主 app 后，跨域由主程序统一配置，可忽略 main.py。
- 大模型失败时接口返回 `success:false`（不会抛 500），不影响其它接口。
- 有问题找 AI 服务这边确认。

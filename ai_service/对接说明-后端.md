# 智能服务对接说明（给后端）

`ai_service` 是独立的 FastAPI 服务（端口 8001），与业务后端（Django，端口 8000）**并行运行、互不挂载**：

```
前端(5173) ──业务请求──→ Django 后端(8000) ──→ MySQL
    └───────AI 请求────→ ai_service(8001) ──→ 通义千问
                              └──图书数据──→ Django /api/books/
```

前端直接调用 8001 的三个 AI 接口（见对接说明-前端.md），后端无需做任何挂载。

## 一、后端唯一的对接点：图书数据

推荐功能需要图书列表。`ai_service/book_repo.py` 已实现：

- 启动后自动请求 **`GET {BACKEND_API_BASE}/books/?page_size=100`**（默认 `http://127.0.0.1:8000/api`，可用环境变量 `BACKEND_API_BASE` 覆盖），按 DRF 分页 `next` 拉全量；
- 字段映射：`id / title / author / category_name → category / description → intro`；
- 结果缓存 5 分钟；
- **后端不可用时自动降级**为本地样例数据（sample_books.json），AI 服务始终可独立演示。

### 请后端配合的一件事
列表接口 `GET /books/` 的返回目前没有 `description`（简介）字段——推荐主要靠简介理解书的内容。请在列表序列化器中补充 `description`（可截断前 200 字），补上后 AI 侧自动生效、无需改代码。

## 二、运行

```bash
pip install -r ai_service/requirements.txt
cp ai_service/.env.example ai_service/.env   # 填 DASHSCOPE_API_KEY
uvicorn ai_service.main:app --port 8001
```

## 三、注意事项

- AI 三接口（/api/ai/*）大模型失败时返回 `success:false`（不抛 500）；
- chat 的会话上下文存内存（TTLCache，30 分钟过期）；多实例部署需换 Redis；
- 图书接口若加了鉴权，请为 AI 服务保留图书列表的匿名读取（或提供专用只读凭证）；
- 有问题找 AI 服务这边确认。

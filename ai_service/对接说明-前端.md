# 智能服务对接说明（给前端）

前端只需要**调用三个接口**，不用关心里面怎么调大模型。所有接口都是 `POST`，收发都是 JSON。

## 一、服务地址

前端配一个 `baseURL` 变量统一管理：

```js
// config.js
export const AI_BASE = 'http://localhost:8001'  // 联调期指向跑 AI 服务的机器；集成后换后端地址
```

- **联调期**：指向运行 AI 服务的机器（本机 `localhost`，跨机用对方局域网 IP）
- **集成后**：挂进后端，用统一后端地址，前缀 `/api/ai` 不变

## 二、三个接口

### 1. 智能摘要 `POST /api/ai/summary`
**用途**：图书详情页的 AI 摘要。两种模式：

**推荐用法（全文安利体，约 200 字）**——传 `book_id`，后端基于全书内容 RAG 生成推荐语：
```json
{ "book_id": 25 }
```
返回（`source`: rag=全文检索生成 / full=短书全文生成 / cached=缓存 / intro=无全文时降级为简介摘要）：
```json
{ "success": true, "summary": "约200字安利体推荐语", "source": "rag", "message": "" }
```
性能：服务启动时已把所有有全文的书**预热进缓存**，正常点击**毫秒级返回**；仅新加的书首次生成需十几秒（建议仍保留 loading 兜底）。无全文且无简介时 `success=false`，按 `message` 提示。

**旧模式（简介一句话摘要）**——传 `text`：
```json
{ "text": "图书简介", "max_length": 60 }
```

### 2. 智能推荐 `POST /api/ai/recommend`
**用途**：用户用自然语言说想看什么，返回推荐书单。用在**搜索/推荐页**或**大模型对话页**。

请求：
```json
{ "query": "想找一本适合周末放松、治愈系的书" }
```
返回：
```json
{
  "success": true,
  "items": [ { "id": 6, "title": "小王子", "reason": "纯净温暖，治愈心灵" } ],
  "reply": "模型原始文本（一般不用，兜底展示时才用）",
  "message": ""
}
```
> `items` 是结构化推荐结果，**优先用它渲染**（每项有书 id / 书名 / 推荐理由）。

### 3. 多轮问答 `POST /api/ai/chat`
**用途**：图书客服对话，能记住上下文。用在**大模型对话页**。上下文由**后端按 `session_id` 自动维护**（30 分钟无活动自动过期），前端不用管历史。

请求：
```json
{ "message": "用户本轮说的话", "session_id": "首次不传，之后带上一次返回的" }
```
返回：
```json
{ "success": true, "reply": "助手的回答", "session_id": "abc123", "message": "" }
```

## 三、统一的成功/失败判断（重要）

**每个接口返回都有 `success` 字段**，前端统一这样处理：

```
success == true   → 显示结果（summary / items / reply）
success == false  → 弹一句 message（"智能服务暂时不可用…"），不要让页面崩
```

这样即使大模型偶尔抽风，页面也稳。

## 四、调用示例（fetch）

前端用 `fetch` 调用即可（项目里已封装在 `book-frontend/src/api/ai.js`，可直接 import 使用）：

```js
async function callAi(endpoint, payload) {
  const res = await fetch(AI_BASE + endpoint, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(payload),
  })
  const data = await res.json()
  if (!data.success) throw new Error(data.message)   // 失败降级
  return data
}

// 用法
const { summary } = await callAi('/api/ai/summary', { text: bookIntro, max_length: 60 })
const { items }   = await callAi('/api/ai/recommend', { query: userInput })
```

## 五、多轮问答的会话怎么维护（用 session_id）

多轮对话的上下文由**后端自动维护**（TTL 会话，30 分钟无活动自动过期），前端**不用攒历史数组**，只需管好一个 `session_id`：

- **第一次**提问：`session_id` 不传（或传空），从返回里拿到后端生成的 `session_id`，存起来；
- **之后每轮**：带上**同一个** `session_id`，AI 就能记住上文；
- **开始新对话**（清空上下文）：换一个新的、或干脆不传，后端会新建。

```js
let sessionId = ''   // 只需存一个会话 id
async function ask(message) {
  const data = await callAi('/api/ai/chat', { message, session_id: sessionId })
  sessionId = data.session_id   // 关键：保存后端返回的会话 id，下轮带上
  return data.reply
}
```

## 六、联调注意

- AI 服务**已开启跨域（CORS）**，H5 直接调不会被拦。
- 接口文档/自测：打开 `AI_BASE/docs`（如 `http://localhost:8001/docs`）。
- 字段以 `schemas.py` / 本文档为准；有疑问找 AI 服务这边确认。
